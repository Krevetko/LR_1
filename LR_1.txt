моя перша репозиторія
В. Що таке Git? 
О. Git - це набір консольних утиліт, які відстежують і фіксують зміни в файлах. З його допомогою можливо порівняти стару версію та нову, проаналізувати, зливати зміни та інше.
В. В якому статусі можуть перебувати файли при роботі з Git?
О. В трьох статусах, а саме збережений у коміті (commited), змінений (modified) та індексований (staged)
В. Для чого використовується команда add? 
О. для початку контролю вибранного файлу
В. Для чого використовується команда status? 
О. Для того щоб дізнатися зміни в робочій директорії
В. Для чого використовується команда commit?
О. Для фіксації змін в реальному часі
В. Для чого використовується команда init?
О. Для створення прихованого каталогу
В. Що необхідно писати в коментарях комітів? 
О. Зміни в файлі
В. Як переглянути список комітів?
О. команда log(git log)
В. Для чого використовується команда checkout?
О. Для переходів між гілками та коммітами
